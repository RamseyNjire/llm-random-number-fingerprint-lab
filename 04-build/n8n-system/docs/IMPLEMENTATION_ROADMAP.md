## Phase 0: Discovery Closeout
- Goal: close critical unknowns before build.
- Deliverables:
  - approved `PROJECT_DEFINE.md`
  - approved `INTEGRATION_REQUIREMENTS.md`
  - initial risk and guardrail matrix
- Exit criteria:
  - all blocking open questions resolved
  - stakeholder sign-off complete

## Phase 1: MVP Happy Path
- Goal: prove one end-to-end workflow path.
- Scope:
  - intake trigger
  - normalization
  - classification/routing
  - one response action
  - one logging destination
- Exit criteria:
  - representative test payload runs successfully
  - output quality and KPI proxy accepted

## Phase 2: Risk Gates and Human-in-the-Loop
- Goal: add trust and escalation controls.
- Scope:
  - low-confidence branch
  - high-risk branch
  - human review queue and handoff
- Exit criteria:
  - risky/ambiguous payloads are safely escalated
  - escalation SLAs and ownership documented

## Phase 3: Ops Hardening
- Goal: make the system maintainable in production.
- Scope:
  - monitoring and alerts
  - rerun and incident procedures
  - data contract verification
  - release and rollback readiness
- Exit criteria:
  - runbook and checklist coverage complete
  - first post-release run verified

## Phase 4: Rollout and Iteration
- Goal: controlled launch and improvements.
- Scope:
  - staged rollout plan
  - KPI tracking cadence
  - backlog of improvements
- Exit criteria:
  - stable run window achieved
  - next optimization cycle prioritized
